## SILENCE CUTTER INSTRUCTIONS

When the user asks to remove silence, pauses, dead air, or filler words:

### REQUIRED STEPS (basic)
1. **extract_audio** - Extract audio track from video
2. **silence_cutter** - Detect silence and filler words, output cut list and trimmed media

### WHEN COMBINED WITH CAPTIONS
If captions are also requested:
1. Run **silence_cutter** first
2. Use **output_video_path** (trimmed video) for later steps
3. Use **output_audio_path** (trimmed audio) for transcription

### MANUAL CUT MODE (EXACT TIMESTAMPS)
If the user provides exact cut ranges (e.g., "cut from 18.005 to 18.670"):
- Use **silence_cutter** with **manual_cut_segments**
- Each segment is an object: {"start": seconds, "end": seconds}
- Treat all manual segments as absolute timestamps on the original source timeline
- Apply all manual segments in one operation/pass (never sequential re-timing after each cut)
- This overrides automatic silence/filler detection
- Do not apply padding unless the user explicitly asks

### WHEN USER DOES NOT SPECIFY CUT MODE
If the user asks to cut/trim/remove parts but does not specify timestamps or auto detection:
- Ask for clarification and provide **two options**:
  1) Auto-cut using silence/filler detection
  2) Manual cut with exact timestamps (e.g., "cut from 18.005 to 18.670")
- Set success=false, add the missing input (e.g., "cut_mode" or "manual_cut_segments")

### KEY PARAMETERS
- **threshold_db**: dBFS threshold for silence (default from config)
- **min_silence_duration**: seconds of silence required to cut (e.g., 0.3s)
- **padding**: seconds to keep around speech to avoid clipping
- **filler_detection**: true/false
- **filler_model_size**: small/base for lightweight local detection
- **filler_language**: auto (supports Arabic, English, and mixed Arabic-English)

### OUTPUTS
- **cut_list_path**: JSON list of cut/keep segments
- **output_video_path**: trimmed video (when provided)
- **output_audio_path**: trimmed audio (when provided)
